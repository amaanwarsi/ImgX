# Contributing to Imgx

Thanks for considering a contribution.

Imgx is intended to be small, understandable, and practical. Contributions that improve reliability, developer experience, extensibility, accessibility, and documentation are welcome.

## Ways to Contribute

- Fix bugs and edge cases
- Improve API clarity
- Add tests
- Improve docs and examples
- Add renderer plugins or renderer improvements
- Improve performance and reduce layout shift
- Refine network-aware behavior

## Development Principles

When contributing, please try to preserve these project goals:

- keep the core lightweight
- avoid unnecessary dependencies
- preserve framework-agnostic behavior
- keep the public API predictable
- prefer extensibility over one-off special cases
- avoid touching images that are not explicitly targeted

## Pull Request Notes

Please aim to:

- keep changes focused
- document any public API changes
- include or update examples when behavior changes
- include tests when practical
- keep code readable and modular

## Documentation Contributions

Docs contributions are especially welcome.

Helpful areas include:

- clearer examples
- framework integration examples
- plugin recipes
- accessibility guidance
- performance notes

## Discussion

If a contribution changes API shape or architecture, it is helpful to open an issue or start a discussion first so the design stays cohesive.
